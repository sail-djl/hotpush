# HotPush 本地启动指南

## 前置依赖

### 必需软件

1. **Python 3.10+**
   - 下载地址：https://www.python.org/downloads/
   - 安装时勾选 "Add Python to PATH"

2. **Node.js 18+**
   - 下载地址：https://nodejs.org/
   - 建议使用 LTS 版本

3. **MySQL 8.0+**
   - 下载地址：https://dev.mysql.com/downloads/mysql/
   - 或使用 Docker：`docker run -d --name mysql -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:8.0`

4. **Redis 7.0+** (可选)
   - 下载地址：https://redis.io/download
   - 或使用 Docker：`docker run -d --name redis -p 6379:7379 redis:7.0`

5. **Git**
   - 下载地址：https://git-scm.com/downloads

### 可选软件

- **Docker Desktop**：用于容器化部署
- **Postman**：API 测试工具
- **Navicat/DBeaver**：数据库管理工具

## 安装步骤

### 1. 克隆项目

```bash
# 克隆仓库
git clone https://github.com/sail-djl/hotpush.git

# 进入项目目录
cd hotpush
```

### 2. 后端环境配置

```bash
# 进入后端目录
cd backend

# 创建 Python 虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 3. 环境变量配置

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，配置以下信息：
```

**.env 文件配置说明：**

```env
# 应用配置
APP_NAME=HotPush
DEBUG=True

# 数据库配置 (必需)
DATABASE_URL=mysql://root:123456@localhost:3306/hotpush

# Redis 配置 (可选)
REDIS_URL=redis://localhost:6379/0

# 管理员账号 (必需)
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_secure_password

# RSSHub 配置
RSSHUB_URL=https://rsshub.rssforever.com

# AI 摘要配置 (可选)
AI_MODEL=gpt-4o-mini
AI_API_KEY=your_api_key
AI_BASE_URL=https://api.openai.com/v1

# 推送渠道配置 (可选)
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id
DISCORD_WEBHOOK_URL=your_webhook_url
```

### 4. 数据库初始化

```bash
# 确保 MySQL 服务已启动

# 创建数据库
mysql -u root -p -e "CREATE DATABASE hotpush CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 数据库表会在首次启动时自动创建
```

### 5. 前端环境配置

```bash
# 进入前端目录
cd ../frontend

# 安装依赖
npm install
```

## 开发模式启动

### 方式一：手动启动 (推荐开发)

**终端 1 - 启动后端：**

```bash
cd backend
# 激活虚拟环境
venv\Scripts\activate  # Windows
# source venv/bin/activate  # macOS/Linux

# 启动后端服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**终端 2 - 启动前端：**

```bash
cd frontend
npm run dev
```

### 方式二：Docker Compose 启动

```bash
# 在项目根目录执行
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 常见问题

### Q: 数据库连接失败

A: 检查以下配置：
1. MySQL 服务是否启动
2. `.env` 文件中的 `DATABASE_URL` 是否正确
3. 数据库用户是否有足够权限
4. 防火墙是否阻止连接

### Q: Redis 连接失败

A: Redis 是可选组件，如果不需要缓存功能，可以在 `.env` 中设置 `REDIS_URL=""` 禁用。

### Q: 前端启动报错

A: 常见解决方案：
1. 删除 `node_modules` 重新安装：`rm -rf node_modules && npm install`
2. 清除 npm 缓存：`npm cache clean --force`
3. 检查 Node.js 版本是否符合要求

### Q: 端口被占用

A: 修改启动命令中的端口号：
```bash
# 后端
uvicorn app.main:app --reload --port 8001

# 前端
npm run dev -- --port 5174
```
