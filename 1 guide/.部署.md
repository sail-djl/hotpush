# HotPush 部署指南

> 最后更新：2026-07-07

## 前置依赖

| 依赖 | 版本要求 | 用途 |
|:--|:--|:--|
| Python | >= 3.11 | 后端运行 |
| Node.js | >= 18 | 前端构建 |
| MySQL | >= 8.0 | 数据库（宿主机独立服务） |
| Redis | >= 6.0 | 缓存（宿主机独立服务） |
| Docker | >= 20.0 | 容器化部署 |

## 后端部署

### 1. 安装依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 配置环境变量

参考 `backend/.env.example` 创建 `backend/.env`：

```env
# 基础配置
DEBUG=false

# 用户认证
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_secure_password

# RSSHub 配置
RSSHUB_URL=https://rsshub.rssforever.com
RSSHUB_FALLBACK_URLS=https://rsshub.feeded.xyz,https://hub.slarker.me
FETCH_INTERVAL_MINUTES=5

# 推送渠道（至少配置一个）
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
DISCORD_WEBHOOK_URL=
WECOM_WEBHOOK_URL=
FEISHU_WEBHOOK_URL=
DINGTALK_WEBHOOK_URL=

# AI 摘要（可选）
AI_MODEL=gpt-4o-mini
AI_API_KEY=

# 数据库
DATABASE_URL=mysql://user:password@localhost:3306/hotpush

# Redis
REDIS_URL=redis://localhost:6379/0
```

### 3. 启动服务

```bash
cd backend
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

- 端口：`8000`
- `--reload` 为开发模式热重载，生产环境去掉

### 4. 验证

```bash
curl http://localhost:8000/api/health
```

#### Docker 配置要点

**docker-compose.yml**（服务器实际配置）

```yaml
services:
  backend:
    build: ./backend
    container_name: hotpush-backend
    expose:
      - "8000"
    extra_hosts:
      - "host.docker.internal:host-gateway"
    environment:
      - TZ=Asia/Shanghai
      - DEBUG=false
      - DATABASE_URL=mysql://root:123456@host.docker.internal:3306/hotpush
      - REDIS_URL=redis://host.docker.internal:6379/0
      - ADMIN_USERNAME=admin
      - ADMIN_PASSWORD=admin123
      - JWT_SECRET=please-change-this-to-a-random-string
      - RSSHUB_URL=http://rsshub:1200
      - FETCH_INTERVAL_MINUTES=5
    volumes:
      - ./data:/app/data
    depends_on:
      rsshub:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - hotpush-network

  frontend:
    build: ./frontend
    container_name: hotpush-frontend
    ports:
      - "3001:80"
    depends_on:
      - backend
    restart: unless-stopped
    networks:
      - hotpush-network

  rsshub:
    image: diygod/rsshub:chromium-bundled
    container_name: hotpush-rsshub
    ports:
      - "1200:1200"
    environment:
      - NODE_ENV=production
      - CACHE_TYPE=memory
      - CACHE_EXPIRE=300
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "curl -f http://localhost:1200 || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 5
      start_period: 30s
    networks:
      - hotpush-network

networks:
  hotpush-network:
    driver: bridge
```

#### 常见问题

**MySQL 连接失败**
- 现象: backend 启动报 `Can't connect to MySQL server`
- 原因: MySQL 运行在宿主机，需要 `host.docker.internal` 映射
- 修复: 确保 docker-compose.yml 中有 `extra_hosts: ["host.docker.internal:host-gateway"]`

**RSSHub 数据源不可用**
- 现象: 热榜数据为空
- 修复: 检查 `RSSHUB_URL` 配置，确认 rsshub 容器健康

**推送渠道配置**
- 至少配置一个推送渠道（Telegram / Discord / 企业微信 / 飞书 / 钉钉）
- 推送渠道在 docker-compose.yml 的 environment 中配置
