# HotPush 基础功能清单

## 功能概述

HotPush 是一个热点聚合推送平台，提供多源热点抓取、智能推送、AI 摘要等核心功能。

## 核心功能模块

### 1. 数据源管理

#### 1.1 内置数据源

- **微博热搜**：抓取微博热搜榜
- **知乎热榜**：抓取知乎热榜内容
- **百度热搜**：抓取百度热搜榜
- **B站热搜**：抓取B站热搜内容
- **V2EX**：抓取V2EX热门话题
- **Hacker News**：抓取Hacker News最佳内容
- **掘金热榜**：抓取掘金技术热榜
- **IT之家**：抓取IT之家热榜
- **少数派**：抓取少数派热门文章
- **NodeSeek**：抓取NodeSeek热门内容
- **豆瓣电影**：抓取豆瓣热映电影
- **豆瓣图书**：抓取豆瓣新书推荐
- **联合早报**：抓取联合早报新闻
- **澎湃新闻**：抓取澎湃新闻热点

#### 1.2 自定义数据源

- 添加自定义 RSS 源
- 编辑自定义源配置
- 删除自定义源
- 启用/禁用数据源

#### 1.3 数据源分类

- 热搜榜（微博、知乎、百度）
- 技术（V2EX、Hacker News、掘金、NodeSeek）
- 科技资讯（少数派、IT之家）
- 视频（B站）
- 影视（豆瓣电影）
- 阅读（豆瓣图书）
- 新闻（联合早报、澎湃新闻）
- 自定义（用户添加的源）

---

### 2. 推送渠道管理

#### 2.1 支持渠道

- **Telegram**：通过 Bot 推送消息
- **Discord**：通过 Webhook 推送消息
- **邮件**：通过 SMTP 发送邮件
- **企业微信**：通过 Webhook 推送消息
- **飞书**：通过 Webhook 推送消息
- **钉钉**：通过 Webhook 推送消息

#### 2.2 渠道配置

- 启用/禁用推送渠道
- 配置渠道参数（Token、Webhook URL 等）
- 测试推送功能
- 查看推送状态

---

### 3. 推送规则管理

#### 3.1 规则类型

- **关键词规则**：根据关键词过滤热点
- **数据源规则**：指定推送的数据源
- **排名规则**：根据排名阈值过滤

#### 3.2 规则操作

- 创建推送规则
- 编辑规则配置
- 启用/禁用规则
- 删除规则

---

### 4. 定时任务调度

#### 4.1 任务配置

- 设置抓取间隔（分钟）
- 配置推送时间
- 设置重试次数
- 配置超时时间

#### 4.2 任务管理

- 启动/停止定时任务
- 手动触发抓取
- 查看任务状态
- 查看任务日志

---

### 5. 用户认证管理

#### 5.1 用户功能

- 用户登录/登出
- 密码修改
- 用户角色管理（管理员/普通用户）

#### 5.2 权限控制

- 管理员：完整权限
- 普通用户：查看权限

---

### 6. AI 摘要生成

#### 6.1 支持模型

- OpenAI（GPT-4o-mini 等）
- Claude
- DeepSeek
- Ollama（本地模型）

#### 6.2 功能特性

- 自动生成热点摘要
- 支持自定义提示词
- 批量生成摘要

---

### 7. 数据统计与分析

#### 7.1 热搜趋势

- 查看各平台热搜趋势
- 热搜排名变化图
- 热度值统计

#### 7.2 推送统计

- 推送成功/失败统计
- 各渠道推送数量
- 推送历史记录

#### 7.3 平台统计

- 各平台热搜数量
- 数据源活跃度
- 抓取成功率

---

### 8. 系统设置

#### 8.1 基础设置

- 应用名称配置
- 调试模式开关
- 语言设置

#### 8.2 抓取设置

- RSSHub 实例配置
- 备用实例列表
- 抓取间隔设置
- 超时时间设置

#### 8.3 缓存设置

- Redis 连接配置
- 缓存 TTL 设置

## 技术实现

### 后端技术栈

- **Web 框架**：FastAPI
- **数据库**：MySQL / SQLite
- **缓存**：Redis（可选）
- **任务调度**：APScheduler
- **RSS 解析**：feedparser
- **HTTP 客户端**：httpx
- **AI 集成**：LiteLLM

### 前端技术栈

- **前端框架**：Vue 3
- **构建工具**：Vite
- **UI 框架**：Tailwind CSS
- **状态管理**：Pinia
- **路由**：Vue Router
- **图表**：Chart.js

## API 接口

### 数据源相关

- `GET /api/sources` - 获取所有数据源
- `POST /api/sources` - 添加自定义源
- `PUT /api/sources/{id}` - 更新数据源
- `DELETE /api/sources/{id}` - 删除数据源

### 推送相关

- `GET /api/push/channels` - 获取推送渠道
- `POST /api/push/channels` - 配置推送渠道
- `POST /api/push/test` - 测试推送
- `GET /api/push/history` - 获取推送历史

### 热点相关

- `GET /api/hotlist/{source}` - 获取热点列表
- `GET /api/trends/{source}` - 获取趋势数据
- `GET /api/trending` - 获取热门条目

### 用户相关

- `POST /api/auth/login` - 用户登录
- `GET /api/users` - 获取用户列表
- `POST /api/users` - 创建用户
