# HotPush 产品设计概述

## 产品定位

HotPush 是一个热点聚合推送平台，旨在帮助用户实时追踪各大平台的热点内容，并通过多种渠道进行推送通知。

## 目标用户

### 主要用户群体

1. **内容创作者**：需要追踪热点话题，获取创作灵感
2. **媒体从业者**：需要实时了解新闻热点
3. **技术开发者**：关注技术社区动态
4. **研究人员**：追踪特定领域热点
5. **营销人员**：了解市场热点趋势

### 用户画像

#### 用户 A：内容创作者

- **需求**：追踪微博、知乎等平台热点
- **痛点**：需要频繁切换多个平台查看热点
- **解决方案**：聚合多平台热点，定时推送

#### 用户 B：技术开发者

- **需求**：关注 V2EX、Hacker News 等技术社区
- **痛点**：工作繁忙，容易错过重要技术动态
- **解决方案**：自定义技术源，推送重要技术文章

## 核心价值

### 1. 信息聚合

- 聚合 14+ 平台热点内容
- 统一展示界面
- 分类管理数据源

### 2. 智能推送

- 多渠道推送（Telegram、Discord、邮件等）
- 自定义推送规则
- 定时推送任务

### 3. AI 增强

- AI 自动生成摘要
- 智能内容过滤
- 热点趋势分析

## 功能架构

```
HotPush
├── 数据源层
│   ├── 内置源（微博、知乎、B站等）
│   ├── 自定义源（RSS）
│   └── 数据源管理
├── 处理层
│   ├── 数据抓取
│   ├── 内容解析
│   ├── AI 摘要

│   ├── MySQL/SQLite
│   └── Redis 缓存

│   ├── 推送渠道
│   ├── 推送规则
│   └── 推送历史
```

## 用户流程

### 1. 首次使用流程

```
注册/登录 → 配置数据源 → 配置推送渠道 → 设置推送规则 → 启动定时任务
```

### 2. 日常使用流程

```
查看热点列表 → 查看趋势分析 → 调整推送规则 → 查看推送历史
```

### 3. 管理流程

```
系统设置 → 用户管理 → 数据源管理 → 推送渠道管理
```

## 界面设计

### 主要页面

1. **热点列表页**：展示各平台热点内容
2. **趋势分析页**：展示热搜排名变化趋势
3. **推送配置页**：配置推送渠道和规则
4. **推送历史页**：查看推送记录
5. **数据源管理页**：管理数据源
6. **系统设置页**：系统级配置
7. **用户管理页**：用户和权限管理

### 设计风格

- **色彩**：深色主题为主，支持浅色主题
- **布局**：左侧导航 + 右侧内容区
- **组件**：使用 Tailwind CSS 组件库
- **响应式**：支持桌面端和移动端

## 数据流设计

### 1. 数据抓取流程

```
定时任务触发 → 调用 RSSHub → 解析 RSS 数据 → 存储到数据库 → 更新缓存
```

### 2. 推送流程

```
定时任务触发 → 获取热点数据 → 应用推送规则 → 调用推送渠道 → 记录推送历史
```

### 3. 用户请求流程

```
用户请求 → API 网关 → 业务逻辑 → 数据库查询 → 返回结果
```
